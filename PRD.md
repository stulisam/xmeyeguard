# 小米电视4 儿童用眼提醒应用 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
**护眼小卫士**（暂定）

### 1.2 产品定位
一款安装在小米电视4（Android TV系统）上的儿童用眼健康提醒应用。通过定时全屏强制提醒+语音播报的方式，帮助3-12岁儿童在使用电视时养成良好的用眼习惯，保护视力健康。

### 1.3 目标用户
| 角色 | 年龄 | 描述 |
|------|------|------|
| 儿童（主要使用者） | 3-12岁 | 日常使用电视看视频、上网课、玩游戏的孩子 |
| 家长（管理者） | 成年人 | 设置提醒规则、管理应用配置的家长 |

### 1.4 核心问题
儿童长时间观看电视屏幕，容易导致视觉疲劳和视力下降。家长不能时刻在旁提醒，需要一个自动化的工具来定时强制提醒孩子休息眼睛。

## 2. 功能需求

### 2.1 核心功能：定时用眼提醒

#### 2.1.1 计时机制
- **使用时长计时**：应用启动后开始计时，累计使用时长达到设定值时触发提醒
- **默认提醒间隔**：每 25 分钟提醒一次（可配置）
- **可配置间隔范围**：15分钟 / 20分钟 / 25分钟 / 30分钟 / 45分钟 / 60分钟
- **计时方式**：后台持续计时，不受用户在电视上的其他操作影响

#### 2.1.2 全屏强制提醒
- 到达设定时间后，弹出**全屏覆盖层**，遮挡所有其他内容
- 覆盖层内容包括：
  - 护眼提示文字："小眼睛，亮晶晶，小朋友，你已经看了25分钟啦，让眼睛休息一下，去看看花花草草吧！"（其中"25分钟"根据实际已使用时长动态替换）
  - 可爱的动画/图标（如卡通眼睛、小动物等）
  - **休息倒计时**：显示剩余休息时间（如"还需要休息 3:00"）
  - 休息结束后自动关闭覆盖层，恢复正常使用
- **强制性**：覆盖层期间，用户无法通过遥控器关闭或跳过（除非输入家长密码）

#### 2.1.3 语音提醒
- 全屏提醒弹出时，同步播放语音提醒
- 语音内容（固定文案）："小眼睛，亮晶晶，小朋友，你已经看了25分钟啦，让眼睛休息一下，去看看花花草草吧！"
- 语音风格：温柔、友好、适合儿童的音色
- 实现方式：预录制音频文件（按此文案录制 ogg/mp3 格式）

#### 2.1.4 休息时长配置
- **默认休息时长**：3 分钟（可配置）
- **可配置范围**：1分钟 / 2分钟 / 3分钟 / 5分钟 / 10分钟
- 休息期间显示倒计时，倒计时结束后自动恢复

### 2.2 家长管控功能

#### 2.2.1 密码保护
- 应用设置页面需要输入**家长密码（PIN码）**才能进入
- PIN码为4-6位数字，首次使用时设置
- 以下操作需要密码验证：
  - 修改提醒间隔时间
  - 修改休息时长
  - 暂停/关闭提醒功能
  - 退出应用
  - 修改密码本身

#### 2.2.2 设置页面（家长模式）
输入正确密码后可访问的配置项：
| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 提醒间隔 | 多久提醒一次 | 25分钟 |
| 休息时长 | 每次休息多长时间 | 3分钟 |
| 语音提醒开关 | 是否播放语音 | 开启 |
| 语音音量 | 提醒语音的音量 | 系统音量 |
| 开机自启动 | 电视开机后自动启动应用 | 开启 |
| 修改密码 | 修改家长PIN码 | - |
| 提醒文案 | 自定义提醒文字内容 | 默认文案 |

### 2.3 应用主界面（儿童视角）

#### 2.3.1 主界面元素
- 简洁友好的界面，适合儿童理解
- 显示内容：
  - 当前已使用时间（如"你已经看了 15 分钟"）
  - 距离下次休息的倒计时（如"还有 10 分钟就要休息啦"）
  - 可爱的卡通形象/图标
- 儿童无法从主界面进入设置页面（需要密码）

#### 2.3.2 最小化/后台运行
- 应用支持最小化到后台运行
- 后台运行时继续计时
- 到时间后自动弹出全屏覆盖层提醒

### 2.4 辅助功能

#### 2.4.1 开机自启动
- 支持设置开机自动启动
- 启动后自动进入计时状态
- 家长可在设置中关闭此功能

#### 2.4.2 使用统计（V2考虑）
- 记录每天使用电视的总时长
- 记录每天休息次数
- 简单的图表展示（可在家长设置页面查看）

## 3. 非功能需求

### 3.1 性能要求
- 应用内存占用不超过 50MB
- 全屏覆盖层弹出延迟不超过 1 秒
- 后台计时精度误差不超过 ±5 秒
- 应用启动时间不超过 5 秒

### 3.2 兼容性
- **目标设备**：小米电视4（Android TV系统）
- **Android版本**：适配 Android 6.0+（小米电视4运行的系统版本）
- **遥控器适配**：所有交互支持小米电视遥控器操作（方向键 + 确认键 + 返回键）
- **屏幕分辨率**：适配 1920x1080 (1080p) 和 3840x2160 (4K)

### 3.3 安全性
- 家长密码本地加密存储（使用哈希+盐值）
- 不收集任何用户数据，不联网
- 纯本地运行的离线应用

### 3.4 用户体验
- 界面字体大：适合电视端远距离观看
- 色彩柔和：避免刺眼的颜色方案
- 交互简单：最少的按键操作
- 动画流畅：提醒覆盖层出现/消失有过渡动画

### 4.3 关键技术点 — 可行性验证结论

#### 4.3.1 全屏覆盖层（SYSTEM_ALERT_WINDOW） ✅ 可行

**结论**：可行，需要一次性 ADB 授权。

- Android TV **没有** 标准 Settings GUI 来授予 `SYSTEM_ALERT_WINDOW` 权限（与手机不同）
- **解决方案**：通过 ADB 命令一次性授权即可永久生效：
  ```
  adb shell pm grant <package_name> android.permission.SYSTEM_ALERT_WINDOW
  ```
- 这是 Android TV 社区的成熟方案，TiviMate 等知名应用均采用此方式
- 授权后，应用可通过 `pyjnius` 调用 `WindowManager` 添加 `TYPE_APPLICATION_OVERLAY` 类型的全屏覆盖窗口
- **对用户的影响**：安装后需要家长通过电脑执行一次 ADB 命令，后续无需再操作。可在安装说明中提供一键脚本

#### 4.3.2 前台服务（Foreground Service） ✅ 可行

**结论**：可行，python-for-android 原生支持。

- python-for-android 内置 Service 支持，在 buildozer.spec 中配置 `services = EyeGuardService:service.py:foreground`
- 前台服务会在通知栏显示一条持久通知（如"护眼小卫士正在守护中"），系统不会杀掉前台服务进程
- 计时逻辑运行在 Service 进程中，即使 Activity 被最小化，计时不中断
- 到时间后，Service 通过 `pyjnius` 调用 `startActivity()` 启动覆盖层 Activity 或通过 `WindowManager` 添加 Overlay
- 额外保险：注册 `AlarmManager` 定时唤醒，防止极端情况下 Service 被回收

#### 4.3.3 按键拦截 ⚠️ 部分可行，需 workaround

**结论**：返回键可完全拦截，HOME 键需用覆盖层方案绕过。

- **返回键（key=27）**：Kivy 通过 `Window.bind(on_keyboard=handler)` 拦截，`return True` 即可阻止默认行为。**完全可行**
- **HOME 键**：Android 系统级事件，普通应用**无法直接拦截**（这是系统安全设计）
- **HOME 键 workaround**：
  - **方案A（推荐）**：使用 `TYPE_APPLICATION_OVERLAY` 全屏窗口，该窗口即使用户按 HOME 键也会保持在最上层显示，因为它不是 Activity 而是系统级覆盖窗口
  - **方案B**：前台 Service 监听前台应用变化（`UsageStatsManager`），检测到用户按 HOME 离开后立即重新拉起覆盖层
- 实际体验：按 HOME 后覆盖层可能闪烁一下但会立即恢复，可接受

#### 4.3.4 开机自启动 ✅ 可行

**结论**：可行，标准 Android 机制。

- 在 buildozer.spec 中声明 `RECEIVE_BOOT_COMPLETED` 权限
- 编写 Java BroadcastReceiver（通过 pyjnius 或自定义 Java 代码）监听 `BOOT_COMPLETED` 广播
- 收到广播后启动前台 Service，Service 再启动 Kivy Activity
- 小米电视 Android 系统对开机自启限制相对宽松（非手机端的严格策略）

#### 4.3.5 遥控器适配 ✅ 可行

**结论**：可行，Kivy 原生支持键盘事件。

- Android TV 遥控器按键映射为标准 Android KeyEvent，Kivy 可通过 `on_keyboard` 事件接收
- 方向键（DPAD_UP/DOWN/LEFT/RIGHT）、确认键（DPAD_CENTER）、返回键（BACK）均可正常捕获
- 需自定义焦点导航系统：Kivy 默认不支持 DPAD 焦点切换，需手动实现按方向键在 Widget 间移动焦点
- 焦点系统实现方案：维护一个可聚焦 Widget 列表，根据方向键计算下一个焦点目标

#### 4.3.6 语音播放 ✅ 可行

**结论**：可行，使用预录制音频文件。

- Kivy 内置 `SoundLoader` 支持在 Android 上播放 wav/ogg 音频
- 采用预录制方案而非 TTS：打包几段预录制的提醒语音文件（ogg 格式，每个约 100KB）
- 避免依赖 TTS 引擎（Android TV 上 TTS 支持不稳定）

### 4.4 技术风险评估总结

| 技术点 | 可行性 | 风险等级 | 说明 |
|--------|--------|----------|------|
| 全屏覆盖层 | ✅ | 低 | ADB 一次性授权，成熟方案 |
| 前台服务 | ✅ | 低 | p4a 原生支持 |
| 返回键拦截 | ✅ | 无 | Kivy 原生支持 |
| HOME 键拦截 | ⚠️ | 中 | 需用 Overlay 窗口绕过，可能有短暂闪烁 |
| 开机自启动 | ✅ | 低 | 标准 Android 机制 |
| 遥控器适配 | ✅ | 中 | 需自定义焦点导航系统 |
| 语音播放 | ✅ | 低 | 预录制音频，SoundLoader 播放 |
| Buildozer 打包 | ✅ | 中 | 环境配置较复杂，建议用 Docker |

**整体结论：Python + Kivy + Buildozer 方案可行，核心功能均有明确实现路径。主要风险在 HOME 键拦截（有 workaround）和遥控器焦点导航（需额外开发），均可解决。**

### 4.6 安装部署流程
1. 通过 U 盘或 ADB 将 APK 安装到小米电视4
2. 通过电脑执行一次 ADB 授权命令（可提供一键脚本）：
   ```
   adb connect <电视IP>
   adb install eyeguard.apk
   adb shell pm grant com.eyeguard.kids android.permission.SYSTEM_ALERT_WINDOW
   ```
3. 首次启动应用，设置家长 PIN 码
4. 后续开机自动运行，无需再操作

## 8. 风险与挑战

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| Kivy在Android TV上的兼容性 | 可能存在显示或性能问题 | 优先在目标设备上测试；备选方案考虑使用Kotlin原生开发 |
| 全屏覆盖层被系统拦截 | Android高版本可能限制悬浮窗 | 申请SYSTEM_ALERT_WINDOW权限；研究TV系统的权限策略 |
| 后台计时被系统杀死 | 计时不准确 | 使用前台服务(Foreground Service)保活；利用AlarmManager做定时唤醒 |
| 遥控器操作体验 | Kivy默认不太适配遥控器 | 自定义焦点管理系统；充分测试遥控器交互 |
| Buildozer打包APK | 环境配置复杂，可能有兼容问题 | 使用Docker环境打包；预先验证打包流程 |
| Python在Android上的性能 | 启动慢、资源占用较高 | 优化代码；控制界面复杂度；考虑预加载 |

## 9. 验收标准

1. **安装**：APK能在小米电视4上正常安装和卸载
2. **基本功能**：应用能准确按设定间隔弹出全屏提醒
3. **语音播放**：提醒时能正常播放语音
4. **强制性**：提醒期间无法通过遥控器跳过（HOME键/返回键被拦截）
5. **密码保护**：没有正确密码无法修改设置或退出应用
6. **配置保存**：修改设置后重启应用，配置不丢失
7. **后台运行**：最小化后仍能正常计时和提醒
8. **开机自启**：电视重启后应用自动启动
9. **遥控器**：所有操作均可通过遥控器完成
10. **稳定性**：连续运行8小时无崩溃
